1️⃣ Daily Follower Count Scraping

Runs: Every day at 9:00 AM

Recommended

What it does:

  • • Connects to Facebook Graph API
  • • Fetches current follower count for all tracked pages
  • • Stores historical data in database
  • • Calculates growth rates week-over-week
  • • Logs results to automation log

📋 Setup Instructions:

  1. SSH into your Fly.io server: ssh root@.fly.dev
  2. Edit crontab: crontab -e
  3. Add this line: 0 9 * * * cd /app && node backend/scripts/scrape-followers.js >> logs/automation.log 2>&1
  4. Save and exit (CTRL+X, then Y, then Enter)

Prerequisites:

  • • Script exists: backend/scripts/scrape-followers.js
  • • Environment variable: FACEBOOK_PAGE_ACCESS_TOKEN
  • • Token permissions: pages_read_engagement

2️⃣ Post Monitoring & Tracking

Runs: Every 6 hours (12:00 AM, 6:00 AM, 12:00 PM, 6:00 PM)

Optional

What it does:

  • • Checks all tracked Facebook pages
  • • Counts new posts since last check
  • • Fetches engagement metrics (likes, comments, shares)
  • • Stores post performance data in database
  • • Calculates velocity scores for trending content
  • • Logs new posts to automation log

📋 Setup Instructions:

  1. SSH into your Fly.io server
  2. Edit crontab: crontab -e
  3. Add this line: 0 */6 * * * cd /app && node backend/scripts/monitor-posts.js >> logs/automation.log 2>&1
  4. Save and exit

Prerequisites:

  • • Script exists: backend/scripts/monitor-posts.js
  • • Environment variable: FACEBOOK_PAGE_ACCESS_TOKEN
  • • Token permissions: pages_read_engagement, pages_manage_posts

3️⃣ Competitor Content Scraping

Runs: Every 3 hours

Advanced

What it does:

  • • Processes all competitor sources in database
  • • Uses Firecrawl to scrape recent posts
  • • Extracts images, captions, engagement data
  • • Stores in scraped_content table
  • • Generates insights automatically
  • • Logs all scraped content

📋 Setup Instructions:

  1. SSH into your Fly.io server
  2. Edit crontab: crontab -e
  3. Add this line: 0 */3 * * * cd /app && node backend/scripts/scrape-competitors.js >> logs/automation.log 2>&1
  4. Save and exit

Prerequisites:

  • • Script exists: backend/scripts/scrape-competitors.js
  • • Environment variable: FIRECRAWL_API_KEY
  • • Competitor sources added in Sources tab

⚠️ Important Notes

  • • Fly.io Note: Use flyctl ssh console to access server
  • • Scripts Status: All automation scripts must be created first
  • • Logs: All cron output goes to logs/automation.log
  • • Testing: Test scripts manually before adding to crontab
  • • Timezone: Cron uses UTC - adjust times accordingly

📘 Facebook Graph API

For automatic posting and page monitoring

Facebook App Configuration

Required for automatic posting, follower scraping, and post monitoring.

📋 Setup Instructions:

  1. Go to Facebook Developers
  2. Create a new app → Select "Business" type
  3. Add "Facebook Login" product
  4. Go to Settings → Basic → Copy App ID and App Secret
  5. Set environment variables: FACEBOOK_APP_ID=your_app_id
    FACEBOOK_APP_SECRET=your_app_secret
    FACEBOOK_PAGE_ACCESS_TOKEN=your_page_token

Token permissions needed: pages_read_engagement, pages_manage_posts, pages_manage_engagement
