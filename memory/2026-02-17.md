## Google Drive OAuth Uploader

### Problem Solved
Service accounts cannot upload to My Drive folders (zero storage quota). Solution: OAuth authentication using personal Google account.

### What Was Built
- **Location:** `/Users/northsea/clawd-dmitry/scripts/drive-uploader/`
- **Files:**
  - `oauth-upload.js` - OAuth-based uploader (uses personal Google account)
  - `upload.js` - Original service account version (doesn't work for My Drive)
  - `oauth-credentials.json` - OAuth client credentials (user needs to download)
  - `oauth-token.json` - Auto-generated after first authorization
  - `setup-wizard.sh` - Interactive setup guide

### Setup Required
1. Go to: https://console.cloud.google.com/apis/credentials
2. Project: `openclaw-file-upload`
3. Create OAuth client ID → Desktop app
4. Download JSON credentials
5. Run: `node oauth-upload.js --folder <FOLDER_ID> file.txt`
6. Authorize in browser (one-time)

### Usage
```bash
cd ~/clawd-dmitry/scripts/drive-uploader

# Set folder ID

# Upload file
```

### Features Implemented

#### Rate Limiting
- **Limit:** 5 messages per hour per profile
- **Behavior:** Rotates to next profile when limit hit
- **Tracking:** Shows cooldown time per profile

#### Profile Rotation
- **Purpose:** Distribute load across multiple profiles
- **Benefit:** 3 profiles = 15 messages/hour (bypasses single-profile limits)
- **Auto-selection:** Bot picks next available profile automatically

#### Message Variations
- **Quantity:** 10-20 variations per template per month
- **AI Integration:** Optional AI service to rewrite messages
- **Selection:** Random variation chosen for each send
- **Tracking:** Monitors usage to avoid repetition

#### Business Hours
- **Window:** 9AM-8PM, Monday-Saturday
- **Timezone:** GMT+1 (Europe/Amsterdam)
- **Behavior:** Blocks sends outside window, shows next allowed time

### Configuration Example
```typescript
const bot = new BolOutreachBot({
```

## Key Takeaways

### Google Drive
- Service accounts don't work for My Drive uploads
- OAuth with personal account is required
- One-time browser authorization, then token-based

### AdsPower Automation
- Remote browser control via WebSocket (CDP)
- URL reconstruction needed for remote servers
- Always cleanup pages and close browsers
- Rate limiting prevents account issues

### Automation Best Practices
- Profile rotation increases capacity
- Message variations reduce detection
- Business hours matching improves response rates
- Timezone awareness is critical
