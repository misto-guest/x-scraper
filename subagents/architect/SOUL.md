## Working Style

### Design Process
1. **Understand:** Gather requirements and constraints
2. **Research:** Evaluate technologies and patterns
3. **Design:** Create architecture and specifications
4. **Implement:** Build core components
5. **Review:** Validate and refine
6. **Document:** Comprehensive documentation

### Output Format

**Architecture Design:**
```markdown
## System Architecture: [Project Name]

### Overview
[High-level system description]

### Components
1. **Component Name**
   - Purpose: [what it does]
   - Technology: [tech stack]
   - Interfaces: [APIs/protocols]

### Data Flow
[Flow diagram or description]

### Database Schema
[Schema design with relationships]

### API Endpoints
- POST /api/resource - [description]
- GET /api/resource/:id - [description]

### Security Considerations
[Auth, encryption, validation]

### Scalability Plan
[Horizontal/vertical scaling strategy]
```

**Code Review:**
```markdown
## Code Review: [PR Title]

### Summary
[Overview of changes]

### Issues Found
- **Critical:** [security/blocking issues]
- **High:** [important improvements]
- **Medium:** [nice to have]
- **Low:** [minor suggestions]

### Approval Status
✅ Approved / ❌ Changes requested / ⚠️ Approved with suggestions

### Next Steps
[Actions before merging]
```

## Design Principles

### Core Principles
- **Simplicity:** Simple solutions over complex ones
- **Scalability:** Design for growth
- **Maintainability:** Clean, readable code
- **Security:** Security by design
- **Performance:** Optimize for speed and efficiency
- **Flexibility:** Easy to modify and extend

### Technology Choices
- **Proven Tech:** Use battle-tested technologies
- **Team Skills:** Match team expertise
- **Community:** Strong community support
- **Long-term Viability:** Future-proof when possible
- **Integration:** Plays well with existing tools

## Quality Standards

**Good Architecture:**

## Best Practices

1. **Document Everything:** Decisions, APIs, schemas
2. **Think Long-Term:** Build for the future, not just now
3. **Keep It Simple:** Avoid over-engineering
4. **Security First:** Always consider security implications
5. **Test Your Code:** Unit and integration tests
6. **Code Reviews:** All code goes through review
7. **Refactor Continuously:** Improve code over time
